# 要件定義
## 一言サービスコンセプト（サービスのキャッチコピーを一言で）
「学びを集めて、未来へシェア」
## 誰のどんな課題を解決するのか？
過去のアプレンティス生のオリジナルプロダクトの記事やチーム開発の記事が散在しており、検索に時間がかかる課題
## なぜそれを解決したいのか？
自分自身が過去のアプレンティス生の情報を検索しても、Qiita、Zenn、Note、ブログなどに散在しており、すべてを効率よく参照できないため、学びの履歴を整理・可視化して後輩や同期が効率的に成長できるようサポートしたい。
## どうやって解決するのか？
過去のアプレンティス生のオリジナルプロダクトの記事やチーム開発の記事をWebアプリで一元管理する。
## 機能要件
### ユーザー機能
- 新規登録・ログイン・ログアウト
- マイページ
    自身の投稿一覧
    自身の記事の編集・削除
    Qiita認証
    Qiita記事の取得・表示
    画像とGitHub URLの追加

### 記事共有機能
- 閲覧画面
    全ての記事を一覧表示
- 詳細画面
    サムネイル画像
    タイトル
    記事のURL
    GitHubのURL
    タグ
- 投稿画面
    サムネイル画像
    タイトル
    記事のURL
    GitHubのURL
    タグ




### 将来的に追加可能な機能（余裕があれば実装）
- **いいね機能**  
  - 記事にいいねを押せる  
- **通知機能**  
  - 通知が来る 
- **ソート機能**  
  - いいね順・公開日順で記事を表示 
- **検索機能**  
  - タグの内容で記事を検索  
- **コメント機能**  
  - 記事にコメントを追加

## 非機能要件
- 個人情報は閲覧できない
- インターフェイスはデザイン崩れしない
- 画面は1秒以内に表示できるようにする
